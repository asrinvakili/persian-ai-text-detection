# Dataset Builder

اسکریپت‌هایی که پیکره `persian-ai-text-corpus` را از صفر تولید می‌کنند. اگر فقط می‌خواهید از پیکره آماده استفاده کنید، نیازی به اجرای این اسکریپت‌ها ندارید — مستقیم به مخزن [`persian-ai-text-corpus`](https://github.com/asrinvakili/persian-ai-text-corpus) مراجعه کنید.

## فهرست اسکریپت‌ها

| اسکریپت | کاربرد | زمان تخمینی |
|---|---|---|
| `extract_wikipedia.py` | استخراج ۱۰۰۰ متن انسانی از ویکی‌پدیای فارسی (نسخه‌های پیش از ۲۰۲۰) | ~۳۰ دقیقه |
| `generate_ai.py` | تولید ۳۰۰۰ متن AI با سه مولد از طریق AvalAI | ~۳ ساعت |
| `build_dataset.py` | ساخت تقسیم train/val/test با لایه‌بندی موضوعی | < ۱ دقیقه |

## ترتیب اجرا

```bash
# ۱. کلید API را در یک فایل بگذارید (هرگز در گیت کامیت نکنید)
echo "YOUR_AVALAI_KEY" > api_key.txt

# ۲. متن انسانی از ویکی‌پدیا
python extract_wikipedia.py

# ۳. متن AI با سه مولد
python generate_ai.py

# ۴. تقسیم‌بندی
python build_dataset.py
```

## نکات طراحی

**چرا نسخه‌های پیش از ۲۰۲۰؟** برای تضمین اینکه متن انسانی پیش از فراگیر شدن LLMها نوشته شده و عاری از آلودگی تولیدی AI باشد.

**تکنیک لنگر موضوع.** برای جلوگیری از خطای توهم در نام‌های خاص فارسی (مثل «نابرابری مارکوف» که LLM ممکن است به «نابرابری مارکف» تغییر دهد)، ۱۵ کلمه نخست متن انسانی در پرامپت به مولد داده می‌شود، ولی اگر مولد عیناً کپی کرد، در پس‌پردازش حذف می‌گردد. این روش تضمین می‌کند که متن انسانی و متن مصنوعی، حتی یک جمله مشترک نداشته باشند.

**تقسیم بر اساس `human_id`، نه نمونه.** چهار نسخه هر موضوع (یک انسانی + سه AI) همگی در یک بخش (train یا val یا test) قرار می‌گیرند، نه پراکنده. تقسیم تصادفی نمونه‌محور منجر به نشت داده و دقت‌های ساختگی بسیار بالا می‌شود.
